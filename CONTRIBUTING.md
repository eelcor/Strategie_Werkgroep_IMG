# Bijdragen aan de VNG AI Kapstok

Welkom! Dit document legt uit hoe u als gemeente of werkgroeplid een bijdrage kunt leveren — zonder technische kennis van Git of GitHub.

---

## Voor gemeenten: document uploaden via de browser

### Stap 1 — Navigeer naar de juiste map

Ga naar de GitHub-pagina van de kapstok en klik u weg naar de map die bij uw bijdrage past.

**Externe invloeden** (rapporten, beleidsdocumenten van buiten):
- Maatschappelijk debat → `00_externe-invloeden/maatschappelijke-ontwikkelingen/`
- Rijksoverheid, NDS, coalitieakkoord → `00_externe-invloeden/overheid-wetgeving/`
- VNG-besluiten, GEMMA → `00_externe-invloeden/vng-ontwikkelingen/`

**Gemeentelijk beleid en ervaringen** → `01_beleid-strategie/[thema]/gemeente-uwgemeente/`

**Uitvoering** → `02_uitvoering/[laag]/gemeente-uwgemeente/`

**Transformatie** → `03_transformatie/[onderwerp]/`

### Stap 2 — Upload uw bestand

1. Klik rechtsboven in de map op **Add file**
2. Kies **Upload files**
3. Sleep uw bestand naar het venster of klik op **choose your files**
4. Wacht tot het bestand is geladen (u ziet de bestandsnaam verschijnen)

### Stap 3 — Geef een beschrijving

Vul bij **"Commit changes"** een korte, duidelijke beschrijving in:

```
Toevoegen: Amsterdam AI-register aanpak (PDF)
```

### Stap 4 — Kies hoe u opslaat

- **Direct naar `main`** → meteen zichtbaar voor iedereen (alleen voor werkgroepleden)
- **Nieuwe branch + Pull Request** → gaat eerst ter beoordeling naar de werkgroep ✓ *aanbevolen voor gemeenten*

Kies **"Create a new branch for this commit and start a pull request"** en geef de branch een naam zoals `bijdrage-amsterdam-ai-register`.

### Stap 5 — Pull Request indienen

Klik op **"Propose changes"**. U wordt doorgestuurd naar een formulier. Vul in:
- **Titel:** korte beschrijving van uw bijdrage
- **Beschrijving:** wat is het document, waarom is het relevant, voor welk thema?

Klik op **"Create pull request"**. De werkgroep ontvangt een melding en beoordeelt uw bijdrage.

## Voor werkgroepleden: beschouwing toevoegen of bijwerken

1. Navigeer naar de README.md van het thema of de laag
2. Klik op het potloodicoon rechtsboven in het bestand
3. Pas de tekst aan
4. Sla op via **"Commit changes"** direct op `main` (werkgroepleden mogen dit)

---

## Bestandsnaamconventie

Gebruik altijd dit formaat — geen spaties, geen speciale tekens:

```
JJJJ-MM_Organisatie_Korte-titel.extensie
```

Voorbeelden:
```
2025-03_Amsterdam_AI-register-aanpak.pdf
2025-02_Rotterdam_Architectuurschets-AI-platform.pdf
2025-01_WRR_Opgave-AI-samenvatting.pdf
2024-11_Rijksoverheid_NDS-gemeenterelevante-passages.pdf
```

---

## Ondersteunde bestandstypen

| Type | Gebruik voor |
|---|---|
| `.pdf` | Rapporten, beleidsdocumenten, presentaties |
| `.docx` | Bewerkbare templates en handreikingen |
| `.md` | Tekst die direct leesbaar is in GitHub |
| `.xlsx` | Templates, checklists, scantools |
